Hexo简明教程（一）

安装Hexo

这是关于hexo的一个极简教程，梳理自己在安装hexo的主要经过，记录遇到的主要问题，一方面算是自己给自己的一份总结，另一方面也期待对他人能够有所帮助。

一、Hexo是什么

Hexo是一个用于搭建blog的简洁高效的框架，支持markdown格式，可以使用一些主题生成漂亮的静态网页。这方面的框架其实有不少，比如Jekyll，不过没使用过，这里就不细说了。

二、安装Hexo

安装Hexo的最好教程是官方文档, 一定要仔细看，基本上常见问题都能够解决。

在安装Hexo之前需要先行安装Node.js和Git，关于这两者的安装方法，在hexo的官方文档有说明，照着做就是了。不过，这里面坑不少，这里重点说说自己遇到的问题。

自己用的MPB15，系统是macOS Sierra，其中Git是自带的，重点就是安装Node.js了。刚开始采用官方提供的安装包直接安装，不过安装好以后没法进一步安装Hexo，折腾好久没解决，于是放弃了这种方法，回头查看文档发现，安装Node.js的最好方法就是用nvm，后来也确实这样安装成功了，看来还是不能自作主张图省事。

安装nvm本来使用hexo官方文档中提供的两个方法（curl命令或wget命令）就可以，不过自己电脑上设置有问题，必须使用SSH协议，所以两种方法都不能用，于是就退回到最原始的方法，采用源码安装，具体方法可参照nvm的文档说明，其中有个小节提供了手动安装（Manual install）的方法，直接安装里面的说明，复制其安装代码就可以搞定了，不过自己的电脑中需要把其中的git clone命令后的地址改成git协议的，否则clone不成功。这里需要提醒的是，一定要完全安装官方文档去做，自己当时就忽视了一个操作，即忘记把以下代码添加到~/.bashrc, ~/.profile, 或 ~/.zshrc文件中去，其实自己电脑中这几个文件都没有，后来是添加到~/.bash\_profile文件中去了，不同版本的系统终端的配置文档不同，这里就需要自己去搞定了。

export NVM\_DIR="$HOME/.nvm"
[ -s "$NVM\_DIR/nvm.sh" ] && . "$NVM\_DIR/nvm.sh" # This loads nvm

安装完nvm之后，使用nvm install stable就可以来安装Node.js了，还是那句话，一定要认真读官方文档，千万不要自作聪明，更不要偷懒。

最后就是安装Hexo了，只要执行npm install -g hexo-cli即可。

这样看来其实安装hexo并不难，可是一旦运气不好，你一定会遇到各种各样稀奇古怪的问题，一般情况下，在官方文档中就能找到答案，实在不行，直接google吧。

此外，虽然自己hexo配置基本成功了，但还是有个问题死活不能解决就是在执行hexo的时候总是跳出下面这个错误信息：

"Error: Cannot find module './build/Release/DTraceProviderBindings'"

虽然这并不影响使用，但看着总是不太舒服。查看nvm在github仓库中的issue，里面提供了几个解决方法，综合看来如下：

先使用npm uninstall hexo卸载hexo，随后使用npm install hexo —no-optional —save重新安装，好多人说这样就解决问题了，可是在自己的电脑上并不管用，暂时没有更好的解决方案，只好先这样吧。

三、安装好Hexo后干啥

很简单，可以开始创建自己的博客了，虽然只能在本地运行，但总算可以迈出第一步了。

在Hexo中创建博客很简单，重点参照官方文档_Setup和官方文档_Writing，大致就是下面这几步：

  1. 进入到你想放置本地博客文件的目录中，随后运行下面几个命令：

    hexo init <folder> #<folder>可以任意命名，用于存放本地博客文件
cd <folder>
npm install

  2. 完成上面几行命令之后，就完成了本地博客的初始化工作，接下来可以进入<folder>\_config.yml中进行一些基本配置，如果不配置也行，后面随时都可以根据自己的需要来进行相关配置。这里暂时采用默认配置。

  3. 运行hexo new <title>，这个命令的作用是在本地新建一篇文章，随后进入<folder>/source/\_posts就可以找到刚刚创建的博客文件，使用你自己喜欢的文本编辑器打开这个文件，就可以在里面写你的博客了。当然，你需要学习一下markdown语法，并且使用markdown语法来写自己的博客。

  4. 完成自己的文章之后，保存并关闭，随后运行下面几行命令，然后根据命令行的提示打开相应网页就可以看看自己的成果了。

    hexo g
hexo s

通过上面几个简单的步骤，我们就完成了博客的创建及生成，并且可以在本地查看基本效果，当然，默认的配置可能不能满足你的需求，那么就需要进一步的设置，这些会在后续的文章中不断涉及，当然，你自己也可以把hexo的官方文档浏览一遍，再次友情提示：你的大部分疑问都可以在那里得到解答。
